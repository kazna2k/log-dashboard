# log-dashboard
log analysis dashboard

## このダッシュボードでできること
今の時点でこれだけ機能しています：

ログファイルをそのままドラッグ＆ドロップで読み込める

総リクエスト数・ユニークIP・エラー率・転送量のサマリーカード

時間帯別アクセス数の折れ線グラフ

**ステータスコード（2xx/3xx/4xx/5xx）**の割合グラフ

上位アクセスURLランキングとバーチャート

最近のログ一覧（ステータスコードの色分け付き）

## 追加された機能
対応ログフォーマットの拡充

自動検出（デフォルト）　ファイルを読み込んだ瞬間に先頭20行を見て Common か Combined かを判定します。検出結果は「✓ 検出フォーマット: ～」と表示されます。

Common Log Format　Nginx・Apache のデフォルト形式で、IPアドレス・時刻・リクエスト・ステータス・サイズの5項目。

Combined Log Format　Common にリファラとユーザーエージェントが加わった形式。Nginx のデフォルトはこちらです。

カスタムフォーマット　「カスタム」を選ぶと入力欄が開きます。Apache の LogFormat ディレクティブと同じ書式でそのまま貼り付ければOKです。たとえば：

```%h %l %u %t "%r" %>s %b "%{Referer}i" "%{User-Agent}i" %D```

のように応答時間（%D）などが入った独自フォーマットにも対応しています。解析できなかった場合はエラーメッセージが表示されます。
